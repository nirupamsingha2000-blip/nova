# Project NOVA Platform — The Future of Science Education

A modern, responsive landing page for Nirupam Science Edge, a premium science coaching institute specializing in concept-based learning and visual understanding.

## 🎯 Overview

Project NOVA is a real multi-page site (not a single scrolling page) showcasing:

- **Home** — Cinematic hero, animated orbital visual, and a live physics simulator
- **About** — Institute mission, vision, and "Why Choose Us"
- **Courses** — Three core programs (Class 11, Class 12 Boards, Foundation), each with its own detail page
- **Results** — Performance metrics with animated counters + testimonials
- **FAQ** — Accordion of common questions
- **Contact** — Demo booking form, map, and contact details
- **Student Portal / Teacher Portal** — Front-end preview dashboards
- **Privacy Policy** — Data handling and third-party processor disclosures
- **AI Assistant** — Floating chat widget backed by a Netlify function + Claude

## 📁 Project Structure

The site is generated from small reusable pieces by `build.js` — **the top-level
`.html` files are generated output**; edit `partials/` or `content/` instead, then
run `node build.js` to regenerate them.

```
Project-Nova/
│
├── build.js                 # Assembles pages from partials/ + content/ (no npm deps)
├── netlify.toml              # Netlify build command + functions directory
│
├── partials/                 # Shared chrome, included on every generated page
│   ├── loader.html           #   cinematic loader + background decor
│   ├── nav.html               #   header/navbar (active link driven by build.js)
│   ├── footer.html            #   footer, newsletter form
│   ├── floating-buttons.html  #   custom cursor, back-to-top, WhatsApp, AI assistant
│   ├── scripts.html            #   shared <script> tags
│   ├── cta-section.html        #   reusable bottom CTA block
│   └── course-cards.html       #   the 3 course cards (shared by home + courses.html)
│
├── content/                   # Per-page unique content, referenced by build.js's page manifest
│   ├── home.html, about.html, courses.html, results.html, faq.html, contact.html
│   ├── privacy-policy.html, student-portal.html, teacher-portal.html
│   └── courses/               #   class-11.html, class-12-boards.html, foundation.html
│
├── index.html, about.html, courses.html, ...   # GENERATED — do not hand-edit
├── sitemap.xml, robots.txt                     # GENERATED by build.js
│
├── netlify/functions/assistant.js   # Serverless function calling the Claude API
│
├── assets/
│   ├── css/
│   │   ├── style.css        # Core styles and component rules
│   │   ├── animations.css   # Animation keyframes and effects
│   │   └── responsive.css   # Mobile and tablet breakpoints
│   │
│   ├── js/
│   │   ├── script.js        # Nav, theme, forms, FAQ, WhatsApp, loader fast-path
│   │   ├── animations.js    # Cinematic loader, scroll reveals, anchor transitions
│   │   ├── portal.js        # Student/Teacher portal page behavior
│   │   ├── concept-lab.js   # Home page's live physics simulator
│   │   ├── ai-assistant.js  # AI assistant chat widget
│   │   └── particles.js     # Particle background effects
│   │
│   ├── images/              # Project images
│   ├── icons/                # Icon assets
│   └── fonts/                # Custom fonts
│
├── README.md                # This file

```

## 💻 Getting Started

1. Edit content under `partials/` or `content/` (never the generated top-level `.html` files directly)
2. Rebuild the generated pages: `node build.js` (or `npm run build`) — Node core only, no dependencies to install
3. Serve the folder with any static server (root-relative paths mean it won't work opened directly as a `file://` URL) — e.g. `python -m http.server 8000` or `npx serve .`
4. Customize colors in `assets/css/style.css` (CSS variables under `:root`)
5. On Netlify, `node build.js` runs automatically as the configured build command — pages regenerate on every deploy

### Backend server (Node.js + Express) for NOVA Mentor

The project includes a lightweight Node.js backend that serves the static site and provides the `/api/nli/explain` endpoint. If you provide an OpenAI API key it will forward requests to OpenAI; otherwise the server returns a mocked response for development.

1. Install dependencies:

```powershell
npm install
```

2. Create a `.env` file in the project root with your OpenAI key:

```
OPENAI_API_KEY=sk-...
```

3. Start the server:

```powershell
npm start
```

The app will be available at `http://localhost:3000`. To preview frontend-only without Node.js you can still use:

```powershell
python -m http.server 8000
```
